Hinweis

Wir empfehlen die Nutzung des Ordners opt jedoch ist das die frei Wahl jedermanns. Innerhalb der Dokumentation bedienen wir uns der einfachheitshalber an Beispielen, die innerhalb des opt Ordner stattfinden.

⚠️ Bitte erfüllen Sie vorab die Voraussetzungen!

Vorbereitung

Führen Sie nun den cUrl Befehl aus.

Bitte achten Sie auf die Version innerhalb des Links!

wget https://updater.err-fire.de/TrafficHandler/beta/0.7.0-Beta/Docker-Compose/traffichandler-docker-0.7.0-Beta.tar --no-check-certificate

Entpacken Sie nun das im Verzeichnis befindliche traffichandler-docker.tar

tar -xvf traffichandler-docker.tar

Sie sehen nun einen weiteren Ordner, mit dem Namen traffichandler. Der Aufbau der Ordnerstruktur spiegelt sich somit wie folgt wieder:

->opt
        ->traffichandler
            ->NGINX
                ->logs
                ->default
                ->conf
                    ->certification
                    nginx.conf
            ->data
                ->traffic_handler
                    ->logs
                    ->database
                    ->data
                    ->conf
                    ->README
            ->docker-compose.yml
            ->LICENSE

Konfiguration

Führen Sie nun die Konfiguration durch. Besuchen Sie dazu den Menüpunkt Konfiguration.

Start des Systems

Damit wäre das System fertig und bereit zum starten. Hierzu folgenden Befehl ausführen.

docker-compose -f /opt/traffichandler/docker-compose.yml up
